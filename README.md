# 社区维修调度台系统

本地社区维修工单调度管理系统，支持前后端分离和 SQLite 本地持久化。

## 技术栈

- **前端**：React 18 + TypeScript + Vite + TailwindCSS + Ant Design 5 + Zustand
- **后端**：Express 4 + TypeScript + ESM + SQLite3
- **认证**：JWT + bcryptjs
- **持久化**：SQLite3 本地文件存储

## 快速开始

### 安装依赖

```bash
npm install
```

### 启动开发服务

```bash
npm run dev
```

这将同时启动：
- 前端开发服务器：http://localhost:5173
- 后端 API 服务器：http://localhost:3001

### 单独启动

```bash
# 仅启动前端
npm run client:dev

# 仅启动后端
npm run server:dev
```

## 数据持久化说明

- 数据库文件位置：`./data/app.db`
- 所有数据（用户、工单、派工记录、状态日志、配置）都存储在此文件中
- 删除此文件将重置所有数据，下次启动时会自动重新创建并插入种子数据
- 重启服务不会丢失任何数据

## 核心业务规则

1. **必填校验**：创建工单时地址和维修类型不能为空
2. **权限控制**：
   - 住户只能查看和创建自己的工单
   - 调度员可以派工、改派、标记完工
   - 管理员可以复核关闭工单、配置系统、导出报表
3. **时间冲突检测**：同一技工在重叠时间段不能被重复派工
4. **操作留痕**：
   - 每次状态变更都记录在 `status_logs` 表
   - 每次派工/改派都记录在 `assignment_logs` 表
   - 记录包含：操作人、操作时间、变更原因

## API 接口说明

### 认证接口
- `POST /api/auth/login` - 登录

### 公开接口
- `GET /api/public/repair-types` - 获取维修类型列表
- `GET /api/public/technicians` - 获取技工列表

### 工单接口
- `GET /api/tickets` - 获取工单列表（按角色过滤）
- `GET /api/tickets/:id` - 获取工单详情（带权限检查）
- `POST /api/tickets` - 创建工单
- `POST /api/tickets/:id/assign` - 派工/改派
- `POST /api/tickets/:id/complete` - 标记完工待复核
- `POST /api/tickets/:id/close` - 复核关闭
- `GET /api/tickets/:id/status-logs` - 获取状态变更日志
- `GET /api/tickets/:id/assignment-logs` - 获取派工历史

### 配置接口（管理员）
- `GET/POST/PUT/DELETE /api/config/repair-types` - 维修类型 CRUD
- `GET/POST/PUT/DELETE /api/config/technicians` - 技工 CRUD
- `GET/POST/PUT/DELETE /api/config/shifts` - 班次 CRUD
- `GET /api/config/check-conflict` - 检查技工时间冲突

### 报表接口（管理员）
- `GET /api/reports/export?status=xxx` - 按状态导出 CSV 报表

## 错误信息说明

系统设计了清晰的错误提示，reviewer 可以直接看懂：

- **前端校验错误**：表单项下方红色提示，如「请输入报修地址」
- **业务逻辑错误**：右上角 message 提示，如「该技工在所选时间段已有派工，请选择其他时间或技工」
- **权限错误**：「无权执行此操作」或「无权查看该工单」
- **参数错误**：「缺少必要参数」
- **服务器错误**：「服务器内部错误，请稍后重试」

所有错误都会在浏览器控制台和后端终端输出详细日志，方便排查。
